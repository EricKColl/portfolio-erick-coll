Arquitectura y fundamentos del sistema

El Producto 2 se ha estructurado siguiendo una arquitectura orientada a la claridad y la mantenibilidad. El patrón MVC organiza la solución, las clases genéricas aportan reutilización, las colecciones dinámicas permiten gestionar información variable en memoria y JUnit verifica el correcto funcionamiento de la lógica de negocio.

// diseno_aplicado
Modelo Entidades, datos y lógica principal del sistema, incluyendo almacenamiento en memoria y operaciones de negocio.
Vista Interacción por consola, menús, entrada de datos y presentación de mensajes al usuario.
Controlador Coordina la comunicación entre vista y modelo, guiando el flujo de la aplicación según la opción elegida.
Java Generics Reutilizar lógica de almacenamiento y acceso sin duplicar código entre distintas entidades del proyecto.
Colecciones dinámicas Gestionar artículos, clientes y pedidos sin fijar de antemano el número de elementos del sistema.
Excepciones + JUnit Refuerzan robustez del programa y comprobación formal de métodos que aplican lógica de negocio.

Secuencia general de trabajo

Esta entrega no solo implementa código: traduce una planificación previa a una solución operativa. El proceso combina análisis del enunciado, configuración técnica del entorno y validación del resultado final.

// proceso_dev
Revisión del enunciado y la rúbrica Se identifican requisitos, restricciones, objetivos evaluables y criterios técnicos que deben cumplirse.
Configuración de Git y entorno de trabajo Se prepara IntelliJ IDEA, el repositorio y el flujo básico de versionado para registrar la evolución del proyecto.
Implementación del modelo estático en Java Se traducen las clases del diseño previo a código real incorporando atributos, métodos, relaciones y lógica asociada.
Aplicación de MVC, colecciones y generics Se estructura la solución con separación de responsabilidades y almacenamiento dinámico según la necesidad del sistema.
Excepciones, validaciones y pruebas Se refuerza la robustez del programa y se comprueban métodos representativos mediante pruebas unitarias con JUnit.
