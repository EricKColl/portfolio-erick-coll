Objetivo y sentido del Producto 4

El objetivo principal de esta entrega consiste en trasladar la persistencia del sistema a una solución basada en ORM, manteniendo la estructura general del proyecto y sustituyendo el acceso tradicional a datos por un modelo más moderno, expresivo y cercano al dominio. Esta fase no supone solo guardar información en una base relacional, sino definir una infraestructura de persistencia coherente, desacoplada y preparada para sostener el funcionamiento completo de la aplicación.

Base conceptual
Persistencia orientada a objetos Las entidades del dominio se conectan con el modelo relacional mediante una capa ORM gestionada por JPA e Hibernate.
Evolución arquitectónica La aplicación conserva MVC e integra una persistencia desacoplada apoyada en DAOs, factorías y utilidades de acceso.
Entorno técnico real El sistema trabaja sobre MySQL y una infraestructura remota mediante Railway, reforzando el valor profesional del proyecto.

Evolución técnica del producto

El Producto 4 representa una progresión clara dentro del proyecto: pasar de una capa de persistencia tradicional a una solución estructurada mediante entidades JPA, relaciones ORM y un sistema de acceso a datos más robusto.

Proceso
Integración de dependencias Se prepara el proyecto Maven con Hibernate, Jakarta Persistence y MySQL Connector como base del nuevo entorno ORM.
Configuración de persistence.xml Se define la unidad de persistencia, la conexión JDBC, el provider de Hibernate y las clases gestionadas.
Creación de la utilidad JPA Se centraliza el EntityManagerFactory para que el resto del equipo disponga de getEntityManager() desde el primer momento.
Mapeo del dominio Las entidades principales se adaptan a ORM, incluyendo herencia en clientes y relaciones persistentes en pedidos.
Consolidación del sistema Se valida la compilación, la ejecución del programa y el funcionamiento global del proyecto tras la integración final.

Persistencia ORM, configuración y fundamentos técnicos

La base técnica del Producto 4 se articula a partir de la configuración de una unidad de persistencia JPA, un proveedor ORM sólido como Hibernate y una utilidad centralizada para gestionar el EntityManagerFactory y generar EntityManager reutilizables desde cualquier parte del sistema.

JPA · Hibernate · EntityManager
Unidad de persistencia Producto4PU define el núcleo de configuración sobre el que se construye toda la interacción entre Java y MySQL.
Provider ORM Hibernate actúa como implementación principal de JPA, traduciendo operaciones del modelo a persistencia relacional.
Registro explícito de entidades Las clases del dominio se declaran manualmente en persistence.xml al mantener exclude-unlisted-classes=true.
Conexión JDBC El archivo persistence.xml define driver, URL, usuario y credenciales necesarias para enlazar el proyecto con MySQL.
Estrategia de esquema La configuración final se mantuvo estable para evitar roturas innecesarias una vez validado el funcionamiento del sistema.
EntityManagerFactory Se centraliza mediante una clase de utilidad para evitar duplicidades y dejar al equipo un getEntityManager() funcional desde el inicio.
Transacciones La integración con DAOFactory y MySQLDAOFactory permite iniciar, confirmar o cancelar operaciones sobre la persistencia.
Preparación de Maven Integración de Hibernate, Jakarta Persistence y MySQL Connector dentro del pom.xml del proyecto.
Configuración de persistence.xml Definición de la unidad de persistencia, propiedades JDBC y registro de entidades del dominio.
Construcción de JPAUtil Implementación de la clase de utilidad tipo Singleton que proporciona EntityManagerFactory y EntityManager.
Integración con la factoría Enlace entre DAOFactory, MySQLDAOFactory y la utilidad JPA para conectar persistencia y arquitectura.
Validación funcional Compilación con Maven, restauración de la ejecución y comprobación completa del sistema sobre la base ORM consolidada.
