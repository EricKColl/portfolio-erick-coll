SECTOR 01

Mission Briefing

CLASIFICACIÓNMigración a GUI

Transformar una aplicación Java orientada a consola en una aplicación de escritorio profesional sin sacrificar la lógica existente. Tres directrices estratégicas guían toda la intervención del Producto 5.

01
M1

De consola a interfaz gráfica

La aplicación original interactuaba mediante menús de terminal. Esta entrega sustituye ese flujo por ventanas, tablas, formularios modales, buscadores y navegación interna construidos con JavaFX, FXML y CSS.

Consola → JavaFX Interacción visual Software de escritorio
02
M2

Preservar la arquitectura

No se reescribe el proyecto desde cero. Se mantiene el patrón MVC, el modelo de datos, las entidades y la capa de persistencia. La intervención se concentra en la vista y los controladores gráficos JavaFX.

MVC intacto Modelo Persistencia JPA
03
M3

Resultado empresarial

El sistema final ofrece un dashboard real con KPIs, módulos de Artículos / Clientes / Pedidos, efectos visuales, sonido sintético y una estructura Maven preparada para mantener y escalar la aplicación.

Dashboard KPIs reales FX visual Maven

MVC como eje

El patrón Modelo-Vista-Controlador separa interfaz, lógica y datos. La vista no contiene reglas de negocio ni SQL: solo coordina eventos y delega.

DAO + Factory

El patrón DAO aísla las operaciones de acceso a datos. Factory centraliza la creación de los DAO, reduciendo acoplamiento y favoreciendo una estructura limpia.

DAO Factory Bajo acoplamiento

SECTOR 10

Incident Log

REGISTROS4 incidencias · 4 resueltas

Log de incidencias detectadas durante la implementación y solución aplicada en cada caso.

⚠ INC.01 · Resuelto

Conflicto de módulos JavaFX

Al lanzar directamente la clase principal aparecía un error de componentes de runtime de JavaFX.

Se introdujo Launcher.java como punto de entrada auxiliar que delega correctamente el arranque del runtime gráfico.
⚠ INC.02 · Resuelto

Carga de estadísticas y estabilidad UI

Calcular indicadores reales podía bloquear el hilo gráfico durante el arranque del dashboard.

DashboardStats se ejecuta en segundo plano mediante Task de JavaFX, consultando agregados sin congelar la interfaz.
⚠ INC.03 · Resuelto

Error FXML por texto iniciado con %

La vista de Clientes mostraba "No resources specified" por un literal que comenzaba con %.

Se reformuló el literal a "PREMIUM (%)" para evitar que FXML interpretara el texto como referencia a recursos externos.
⚠ INC.04 · Resuelto

Distribución inicial del dashboard

Algunas tarjetas no aprovechaban bien el espacio y se percibía solapamiento visual.

Se eliminó un panel redundante de accesos rápidos, se amplió la cabecera principal y se reorganizó la jerarquía visual.
