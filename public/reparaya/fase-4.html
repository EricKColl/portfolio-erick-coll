SECCIÓN · 02

Arquitectura del Sistema

La solución se apoya en una arquitectura dual: WordPress sirve el contenido y el bloque personalizado consume el endpoint REST publicado en el Producto 3, dentro del servidor UOC y replicado en entorno local con Docker.

flujo de datos

De WordPress al endpoint JSON

01 Visita pública

El usuario accede a /~uocx3/ y WordPress resuelve la plantilla FSE correspondiente.

02 Render del tema

El tema reparaya-producto-4 compone cabecera, plantilla y pie con block theme nativo.

03 Bloque PHP

En la página Nuestros servicios se carga el bloque Genesis con su lógica block.php.

04 Consumo REST

wp_remote_get() ataca el endpoint del Producto 3 y valida el JSON recibido.

05 Render dinámico

Se muestra el total global y un grid de zonas con número de servicios y porcentaje.

capas del sistema

Stack técnico operativo

Cada capa cumple una función concreta dentro del flujo. La separación facilita el doble despliegue local / UOC sin tocar la lógica del tema ni del bloque personalizado.

PROTOCOLO · Ω

Quantum Block

Bloque PHP personalizado creado con Genesis Custom Blocks, capaz de consumir el web service REST del Producto 3 y renderizar el listado de servicios agrupados por zona dentro de cualquier página WordPress.

creación del bloque

Pasos del bloque personalizado

01 Activar Genesis Custom Blocks

Plugin instalado, accesible desde el panel admin de WordPress.

02 Definir bloque servicios-zonas

Slug, icono, atributos (endpoint) y editor preview configurados.

03 Lógica en block.php

Cadena de endpoints (local → UOC), wp_remote_get() y validación del JSON.

04 Render server-side

Bloque devuelve HTML con total global, grid de zonas y porcentaje por zona.

05 Inserción en página

Se añade desde el editor a la página Nuestros servicios y queda persistido.
